# Спецификация: «Запись на звонок»

Упрощённый сервис бронирования времени по мотивам Cal.com.

## Общее описание

Владелец календаря публикует типы событий с доступными слотами, гости выбирают свободное время и записываются на звонок. Без регистрации, авторизации и интеграций с внешними календарями.

## Роли

### Владелец календаря
- Один захардкоженный профиль (аватар, имя, подпись "Host")
- Используется по умолчанию в админской части
- Может:
  - Создавать, просматривать, редактировать и удалять типы событий
  - Просматривать список предстоящих встреч

### Гость
- Без регистрации и авторизации
- Может:
  - Просматривать каталог типов событий
  - Выбирать тип события, дату и свободный слот
  - Создавать бронирование (имя + email)
  - Просматривать и отменять свои бронирования (хранятся в LocalStorage)

## Модели данных

### EventType (тип события)

| Поле | Тип | Описание |
|------|-----|----------|
| id | string (uuid) | Уникальный ID |
| name | string | Название |
| duration | enum (10, 15, 20, 30) | Длительность в минутах |
| createdAt | datetime (UTC) | Дата создания |

### Booking (бронирование)

| Поле | Тип | Описание |
|------|-----|----------|
| id | string (uuid) | Уникальный ID |
| eventTypeId | string (uuid) | FK на тип события |
| guestName | string | Имя гостя |
| guestEmail | string | Email гостя |
| startTime | datetime (UTC) | Начало слота |
| endTime | datetime (UTC) | Конец слота (startTime + 30 мин) |
| createdAt | datetime (UTC) | Дата создания |

## Бизнес-правила

### Слоты
- Фиксированная длительность слота: 30 минут
- Рабочие часы: 09:00–17:00, каждый день (пн–вс)
- Доступные слоты: 09:00, 09:30, 10:00, ..., 16:30
- Окно записи: 14 дней от текущей даты

### Встречи и занятость
- Длительность встречи может быть меньше слота (10, 15, 20 мин), но встреча занимает весь слот
- Встреча начинается во время начала слота
- Глобальная занятость: если слот занят — он занят для всех типов событий
- Уникальность startTime среди всех бронирований (unique constraint)

### Удаление
- Отмена бронирования гостем: запись удаляется физически из базы
- Удаление типа события: каскадное — удаляется тип + все его бронирования

### Валидация (при создании бронирования)
- Слот в пределах 14 дней от текущей даты
- Слот не занят (проверка на бэкенде)
- startTime кратен 30 минутам, в пределах рабочих часов
- Имя и email не пустые

## API endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/event-types` | Список типов событий |
| POST | `/api/event-types` | Создать тип события |
| PATCH | `/api/event-types/:id` | Обновить тип события |
| DELETE | `/api/event-types/:id` | Удалить тип события (каскадно с бронированиями) |
| GET | `/api/slots?date=&eventTypeId=` | Свободные слоты на дату |
| POST | `/api/bookings` | Создать бронирование |
| GET | `/api/bookings/:id` | Получить бронирование по ID |
| DELETE | `/api/bookings/:id` | Отменить бронирование |
| GET | `/api/admin/bookings` | Все предстоящие встречи |

## Страницы и навигация

### Навигация (header)
Логотип "Calendar" | "Записаться" | "Админка"

### Роуты

| Роут | Страница | Описание |
|------|----------|----------|
| `/` | Главная | Лендинг с описанием сервиса, кнопка "Записаться" |
| `/booking` | Каталог типов | Профиль владельца + карточки типов событий |
| `/booking/:eventTypeId` | Бронирование | Инфо о событии, календарь, слоты, форма бронирования |
| `/admin` | Предстоящие встречи | Список будущих бронирований (приоритетная страница админки) |
| `/admin/event-types` | Типы событий | CRUD типов событий |

### Процесс бронирования (flow)

1. `/booking` — гость видит профиль владельца + карточки типов событий
2. Клик на карточку → `/booking/:eventTypeId`
3. Выбирает дату в календаре (14 дней)
4. Видит слоты на выбранную дату с статусами "Занято"/"Свободно"
5. Выбирает свободный слот → "Продолжить"
6. Заполняет форму: имя + email
7. Бронирование создано → сообщение "Успешно" → редирект на `/booking`
8. ID бронирования сохраняется в LocalStorage

### Компонент "Мои записи" (гость)
- Читает ID бронирований из LocalStorage
- Запрашивает данные по каждому ID
- Показывает список с возможностью отмены

- Touch-friendly: минимум 44px tap targets
- Tailwind responsive утилиты (sm:, md:, lg:)

- Все даты хранятся и передаются в UTC
- Фронтенд конвертирует в локальное время пользователя
