# E2E-тестирование: дизайн-спецификация

## Цель

Проверить основной пользовательский сценарий бронирования в реальном браузере — от просмотра каталога типов событий до успешного создания бронирования. Фронтенд и бэкенд работают вместе.

## Решения

- **Инструмент**: Playwright
- **Объём**: один сценарий — основной путь бронирования (happy path)
- **Серверы**: Playwright `webServer` автоматически стартует backend и frontend
- **Данные**: seed-данные бэкенда (`event-types.jsonl`, `bookings.jsonl`), перезапуск при каждом прогоне
- **Подход**: один линейный тест с `test.step()` для группировки шагов

## Структура файлов

```
e2e/
├── package.json            # @playwright/test, typescript
├── playwright.config.ts
├── tsconfig.json
└── tests/
    └── booking-flow.spec.ts
```

Отдельный `package.json` — e2e изолирован от frontend/backend. Совместимо с существующим Makefile-таргетом `make test-e2e` (`cd e2e && npx playwright test`).

## Конфигурация Playwright

| Параметр | Значение |
|----------|----------|
| baseURL | `http://localhost:5173` |
| webServer | backend (port 3000) + frontend (port 5173) |
| Браузер | Chromium only |
| Timeout | 30 000 ms |
| Retries | 0 |
| Screenshots | only-on-failure |

### webServer

Два сервера запускаются автоматически:

1. **Backend**: `npm run dev` из `../backend`, port 3000, `reuseExistingServer: !process.env.CI`
2. **Frontend**: `npm run dev` из `../frontend`, port 5173, `reuseExistingServer: !process.env.CI`

## Тестовый сценарий

Файл: `tests/booking-flow.spec.ts`

Один тест: **"гость бронирует слот"**

### Step 1: Каталог типов событий

- `page.goto('/booking')`
- Проверить: карточки типов событий видны на странице
- Кликнуть на первую карточку
- Проверить: URL изменился на `/booking/:eventTypeId`

### Step 2: Выбор даты и слота

- Проверить: календарь отображается
- Кликнуть на доступную дату (сегодня)
- Проверить: список слотов появился
- Кликнуть на первый свободный слот (статус "Свободно")
- Нажать кнопку "Продолжить"

### Step 3: Заполнение формы и бронирование

- Проверить: модальное окно / форма бронирования открылась
- Заполнить поле "Имя"
- Заполнить поле "Email"
- Нажать кнопку "Забронировать"
- Проверить: сообщение об успешном бронировании отображается

## Данные для тестов

Бэкенд при старте загружает seed-файлы:
- `backend/data/event-types.jsonl` — типы событий (используем первый из списка)
- `backend/data/bookings.jsonl` — существующие бронирования

Playwright `webServer` перезапускает бэкенд перед каждым прогоном — in-memory store пересоздаётся с чистыми seed-данными.

## Интеграция

- **Makefile**: существующий таргет `make test-e2e` работает без изменений
- **Pre-commit (`make check`)**: e2e НЕ включается — слишком медленно для хука
- **CI**: можно добавить отдельным шагом в GitHub Actions (вне скоупа этой задачи)
